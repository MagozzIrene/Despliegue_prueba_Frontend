Este proyecto es una aplicación full-stack estilo WhatsApp desarrollada con React, Node.js, Express y MongoDB. El frontend está construido con React y Vite, utiliza Context API para el manejo global de estado (autenticación, contactos, mensajes y grupos) y está organizado en componentes reutilizables y hooks personalizados para separar responsabilidades y evitar duplicación de lógica. La app es completamente responsive, funcionando en un rango de pantallas desde 320px hasta 2000px, con vistas adaptadas a mobile y desktop (incluyendo diseño de panel dividido en escritorio y navegación por pantallas en móvil). Se implementó enrutamiento con react-router, incluyendo rutas dinámicas con parámetros. Tanto el registro como el login poseen validaciones y manejo de errores. Se usa localStorage para almacenar de manera segura el token JWT y los datos del usuario autenticado. La interfaz incorpora filtros, buscadores independientes por pestaña, previews de mensajes, renderizado optimizado, formateo dinámico de fechas (Hoy, Ayer, fecha completa) y separadores de fechas dentro del chat. También se incorporaron loaders, mensajes vacíos, placeholders y un diseño accesible con contrastes adecuados.

A nivel funcional, el usuario puede registrarse, iniciar sesión, cambiar contraseña y cerrar sesión. Puede buscar contactos, enviar y recibir solicitudes, aceptarlas o rechazarlas, eliminarlas y ver la última conexión de cada usuario junto con su estado online actualizado mediante una lógica de ping al backend. En los chats individuales se incluye sistema de “visto” con doble tilde azul. En los grupos se puede crear grupos, ver miembros, enviar mensajes, invitar participantes y visualizar el estado de lectura: los mensajes grupales muestran el tilde azul únicamente cuando todos los demás miembros lo leyeron. También se optimizó la visualización de mensajes consecutivos del mismo remitente evitando repetir avatar y nombre innecesariamente. Se pueden eliminar mensajes, contactos, solicitudes y grupos, todo con modal de confirmación para evitar acciones accidentales. Existe un usuario Invitado con datos precargados para facilitar pruebas.

El backend está desarrollado con Node.js y Express siguiendo una arquitectura en capas compuesta por rutas, controladores, servicios, repositorios y modelos. Las rutas están organizadas por dominios (auth, usuarios, contactos, grupos, mensajes) y protegidas mediante JWT. Se utiliza bcrypt para el hash de contraseñas, JSON Web Tokens para la autenticación, y variables de entorno para proteger claves sensibles. Se integra Nodemailer para envío de correos reales de verificación de cuenta. La base de datos es MongoDB Atlas y se estructura mediante varios modelos: Usuario, Contacto, Solicitud, Mensaje, Grupo, Miembro de grupo, Invitación y Mensaje de grupo con su campo de lectura por usuario. Los servicios encapsulan la lógica de negocio completa: manejo de contactos, validaciones, invitaciones, verificación de permisos de administrador, registro de actividad, estado de lectura, presencia online y sincronización de mensajes. Los repositorios son responsables de toda interacción con MongoDB mediante Mongoose, asegurando que los controladores permanezcan limpios y que la lógica se mantenga desacoplada.

La aplicación cumple el criterio de rutas privadas, protección de endpoints, envío de correo real, validaciones, seguridad y persistencia. Frontend y backend están desplegados de manera independiente en Vercel, y el proyecto está preparado para ser probado tanto con usuarios reales como con el usuario Invitado. A nivel técnico, el proyecto demuestra manejo de estados complejos, sincronización entre múltiples entidades, diseño escalable, y una integración completa entre servidor y cliente. A pesar de que quedaron algunas funciones sin implementar en el frontend (como la edición de grupos o la opción de salir del grupo), la base funcional es sólida y cumple ampliamente los requisitos del TP, incorporando además características adicionales como estado online, formateo inteligente de fechas, sistema de lectura individual y grupal, agrupamiento de mensajes y un flujo de interacción completo entre usuarios.
